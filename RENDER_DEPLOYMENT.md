# Render Backend Deployment

## Render service

Create a new Blueprint on Render from this repository. The root `render.yaml` defines a Docker web service for `MeetingRoomBookingSystem`.

Required environment variables:

- `SPRING_DATASOURCE_URL`: JDBC URL for your MySQL-compatible database, for example `jdbc:mysql://host:4000/test?sslMode=REQUIRED&serverTimezone=UTC`
- `SPRING_DATASOURCE_USERNAME`: database username
- `SPRING_DATASOURCE_PASSWORD`: database password
- `CORS_ALLOWED_ORIGINS`: your deployed Vercel frontend URL, for example `https://your-app.vercel.app`
- `ADMIN_NAME`: initial admin display name
- `ADMIN_EMAIL`: initial admin email
- `ADMIN_PASSWORD`: initial admin password
- `APP_TIME_ZONE`: timezone for booking status calculations, for example `Asia/Kolkata`

`JWT_SECRET` is generated by Render from the blueprint.

## Vercel frontend

After the Render service deploys, set this Vercel environment variable and redeploy the frontend:

- `VITE_API_URL`: `https://your-render-service.onrender.com`

The backend exposes `GET /health` for Render health checks.

## TiDB Cloud example

Use the public TiDB host and keep the password in `SPRING_DATASOURCE_PASSWORD`, not inside the JDBC URL:

```text
SPRING_DATASOURCE_URL=jdbc:mysql://gateway01.ap-southeast-1.prod.alicloud.tidbcloud.com:4000/test?sslMode=REQUIRED&serverTimezone=UTC
SPRING_DATASOURCE_USERNAME=your_tidb_username
SPRING_DATASOURCE_PASSWORD=your_tidb_password
APP_TIME_ZONE=Asia/Kolkata
```

If your app database does not exist yet, create it in TiDB SQL editor:

```sql
CREATE DATABASE bookMeetingRoom;
```
